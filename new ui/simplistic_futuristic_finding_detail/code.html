Vulnerability Description

expand_more

A SQL Injection vulnerability was identified in the application. This flaw allows an attacker to interfere with the queries that an application makes to its database. It can be used to view data they are not normally able to retrieve, or even modify or delete this data, causing persistent changes to the application's content or behavior. This instance was found in the 'productID' parameter on the main search page.

Evidence

Proof of Concept

The payload ' OR '1'='1 was injected into the id parameter, causing the database to return all product records instead of just one, confirming the vulnerability.

Remediation Steps

  1. Use parameterized queries (prepared statements) to prevent attacker input from being interpreted as SQL commands.
  2. Implement input validation to ensure that user-supplied data conforms to the expected format and type.
  3. Apply the principle of least privilege by restricting database user permissions to only what is necessary.
